Overview

What is EQI?

The Engineering Quality Index is a centralized view of quality, security and test coverage across your repositories and projects. It reads the reports your tools already generate, scores each category automatically, and rolls everything up into a single index you can track over time — so engineering health is something you can actually see.

This isn’t a compliance checkbox

EQI exists so that good engineering work gets seen — and gaps get fixed before clients find them.

Fewer production fires

Catch dependency vulnerabilities and misconfigurations before they ship — not after a client reports them.

Recognition that’s earned

A repo climbing from D to B is visible proof of work — to your lead, your PM, and leadership.

Client trust, quantified

“90% Application Security” is a stronger story than “we think it’s fine.”

Know exactly what to fix next

No more guessing — the lowest-scoring category tells you exactly where to focus this sprint.

Ways to generate EQI

Levels

Levels & categories

EQI operates at two levels. Repository scores measure an individual codebase; project scores measure attributes across the whole project. Repository scores roll up to contribute to the overall index.

Repository level

Quality & security of an individual repository

  • Code Quality
  • Dependency Security
  • Application Security
  • Mobile Security
  • Observability & Runtime Quality

Project level

Attributes that apply across the whole project

  • Infrastructure Security
  • Test Coverage & Effectiveness

Project-level reports must be uploaded from the Project Dashboard, and these scores feed directly into the overall Engineering Quality Index.

Manual upload guide

How to upload report

EQI supports manual uploads at both project and repository level. Pick the level that matches your report, then follow the steps below.

Want detailed steps with screenshots? See the detailed documentation for a full visual walkthrough.

Project level

From the Project Dashboard

For Infrastructure Security and Test Coverage & Effectiveness.

  1. Open the Project Dashboard

    Navigate to the project you want to score.

  2. Open the category tile

    Click the Infrastructure Security or Test Coverage & Effectiveness card.

  3. Add a report

    Click Add Report and select the appropriate tool.

  4. Choose your file

    Pick the report that matches the tool’s supported format.

  5. Upload & wait

    Click Upload and give it a few seconds to process and generate.

Repository level

From inside a repository

For Code Quality, Dependency Security, Application Security, Mobile Security and Observability.

  1. Open the project

    Then navigate to the repository you want to score.

  2. Add a report

    Click Add Report within that repository.

  3. Select the tool

    Choose the tool that produced your report.

  4. Upload the file

    Select the report file in its supported format.

  5. Wait for processing

    EQI parses the file and updates the repository’s category score.

Always upload from the right level. Project reports belong on the Project Dashboard; repository reports belong inside the relevant repository. Uploading at the wrong level will not produce the score you expect.

How it works

From raw report to a single index

EQI follows the same path whether a report arrives automatically or by hand. Each stage is normalized, so scores stay comparable across tools and teams.

1

Run your tools

SonarQube, Trivy, OWASP ZAP, Jest and others produce their usual reports.

2

Bring reports in

Automatically through CI/CD, or by uploading a supported file manually.

3

Parse & extract

EQI reads each report and pulls out findings, severities and coverage data.

4

Score per category

Each supported category receives a normalized 0–100 score.

5

Roll up

Category scores combine into repository scores, then into the overall project EQI.

6

Monitor

Track scores and trends on the dashboard and drill into the underlying findings.

Processing usually takes a few seconds. Larger reports take a little longer, so give the page a moment to finish generating before refreshing.

Interpreting results

Reading the scores

Every category is scored on the same 0–100 scale, where higher is healthier. The index reads like a “greater-than-or-equal” check: the goal is to keep each category at or above your team’s bar.

Best practices

Getting the most out of EQI

Automate wherever you can

Wire reports into CI/CD so scores refresh on every build. Manual upload is best kept for client work and one-off audits.

Match the format exactly

Each tool expects a specific format. Confirm your file is the supported JSON, HTML or PDF before uploading.

Upload at the right level

Project reports go on the Project Dashboard; repository reports go inside the repository. Mixing them up skews the index.

Keep tool versions current

Reports from supported tool versions parse reliably. If generation fails, an outdated or non-standard report is the usual cause.

Read trends, not just snapshots

A single score is a moment in time. Watch the direction after refactors, dependency bumps and new test suites.

Triage by severity and category

Use the category breakdown to fix the highest-impact issues first instead of chasing every finding at once.

FAQ

Frequently asked questions

Do I need CI/CD to use EQI?

No. The automated pipeline is the smoothest option, but you can upload reports manually at any time. Manual upload is designed exactly for client projects and environments where CI/CD isn’t integrated.

Which file should I upload?

Upload the report in the format listed for your tool in the Supported tools table — JSON, HTML or PDF depending on the tool. If a format isn’t listed for a tool, EQI can’t parse it.

Where do project vs repository reports go?

Project-level reports (Infrastructure Security, Test Coverage & Effectiveness) are uploaded from the Project Dashboard. Repository-level reports are uploaded from inside the relevant repository.

My report failed to process — what should I check?

Confirm three things: the file matches the tool’s supported format, it was generated by a supported tool version, and you’re uploading at the correct level. Those cover the large majority of failures.

How does a repository score affect the project?

Repository-level scores roll up and contribute to the overall project score, alongside the project-level categories. Improving a weak repository raises the overall Engineering Quality Index.

Are scores comparable across different tools?

Within a category, yes — EQI normalizes each report to the same 0–100 scale so a score means the same thing regardless of which supported tool produced the underlying report.

How often should I generate reports?

For automated repositories, on every build or merge keeps scores current. For manual uploads, a regular cadence — such as per release or per audit — keeps trends meaningful.

Why does processing sometimes take longer?

Processing time scales with report size. Most reports finish in a few seconds; large reports take a little longer. Wait for generation to complete before refreshing.
